# <span className="soft">atom.io</span>/foundations/subject

Subjects are small observable primitives.

Use `Subject` when you want keyed subscribers and explicit `next` calls. Use
`StatefulSubject` when subscribers also need the latest state stored on the subject.

## Subject

`subscribe(key, subscriber)` registers a subscriber under a string key and returns an
unsubscribe callback.

`next(value)` calls all current subscribers with that value.

## StatefulSubject

`StatefulSubject` extends `Subject` with a public `state` property.

When you call `next(value)`, `state` updates before subscribers are notified.
